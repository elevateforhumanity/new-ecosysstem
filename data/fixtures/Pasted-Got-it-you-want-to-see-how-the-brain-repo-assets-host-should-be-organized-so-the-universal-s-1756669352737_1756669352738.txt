Got it ✅ — you want to see how the “brain” repo (assets host) should be organized so the universal script can pull everything in for all your sister sites.

Here’s a clean, future-proof folder structure:


---

📂 Brain / Assets Repo Structure

/ui
   header.html           ← your universal header (nav, logo, dropdowns)
   footer.html           ← your universal footer (sitemap, contact info)
   styles.css            ← global styles (optional)

 /config
   health-programs.json  ← healthcare programs list (state-funded + open)
   beauty-programs.json  ← beauty/esthetics programs list
   partners.json         ← partner directory (for affiliate / 50-50 splits)
   payments.json         ← any global payment or API config

 /scripts
   efh-universal.v1.js   ← the universal script you drop on every site

 /branding
   logo.png
   favicon.ico
   brand-guide.pdf

 index.html              ← (optional landing page for the assets repo)


---

🔗 How the Universal Script Pulls From Here

Header/Footer:
https://assets.elevateforhumanity.org/ui/header.html
https://assets.elevateforhumanity.org/ui/footer.html

Programs Config:
https://assets.elevateforhumanity.org/config/health-programs.json
https://assets.elevateforhumanity.org/config/beauty-programs.json

Partners Directory (optional future use):
https://assets.elevateforhumanity.org/config/partners.json

Script Itself:
https://assets.elevateforhumanity.org/scripts/efh-universal.v1.js



---

🛠️ On Each Sister Site

You only add:

<div id="efh-header"></div>
<div id="efh-programs"></div> <!-- optional -->
<div id="efh-footer"></div>

<script src="https://assets.elevateforhumanity.org/scripts/efh-universal.v1.js"
        data-assets="https://assets.elevateforhumanity.org"
        data-payapi="https://pay.elevateforhumanity.org/api/checkout"
        data-programs="/config/health-programs.json"
        defer></script>

Change data-programs depending on which catalog the site needs (health, beauty, staffing, etc.).


---

✅ With this setup:

Update once in /ui/header.html → every site updates.

Add/remove a program in /config/health-programs.json → instantly reflects everywhere.

Add a new partner in /config/partners.json → universal script can display it across the ecosystem.
